# Token Usage

Humbug tracks token usage across all AI conversations within a mindspace. The **Token Usage**
tab gives you a per-model breakdown of how many tokens have been consumed, so you can understand
which models and providers are driving cost.

---

## Opening the Token Usage tab

Open the Token Usage tab from **Mindspace → Token Usage**, or with **Ctrl+Shift+U**
(macOS: **Cmd+Shift+U**).

The tab is mindspace-scoped: it shows usage for the currently open mindspace only. If no
mindspace is open, the tab displays an empty state.

---

## What is tracked

Each time an AI completes a response, Humbug records the token usage reported by the provider
for that (provider, model) pair. The following token categories are tracked:

| Category | What it means |
|---|---|
| **Input** | Prompt and context tokens sent to the model |
| **Output** | Tokens generated by the model in its response |
| **Cache hits** | Tokens read from the provider's prompt cache (shown only if non-zero) |
| **Cache writes** | Tokens written to the provider's prompt cache (shown only if non-zero) |

Usage data is stored in `.humbug/usage.json` within the mindspace and persists across sessions.

---

## The usage dashboard

### Hero card

The top of the dashboard shows the **total token count** (input + output) across all models,
along with a summary line breaking down total input vs output tokens and the number of
providers and models used.

### Token mix

A set of cards showing each token category with its count and a short description.

### Details

This section lists usage grouped by provider, with each provider shown as a card. Within each
provider card:

- Each model is listed with its display name and total token count
- A breakdown line shows input, output, cache hit, and cache write token counts
- A coloured bar shows the model's share of total token usage relative to all other models

Providers and models are sorted by total token usage (highest first), making it easy to see
which models are consuming the most tokens.

---

## Resetting usage

The **Reset Usage** button at the bottom of the dashboard clears all accumulated usage data for
the current mindspace. This is useful if you want to start tracking from a clean slate — for
example, after switching to a different model or starting a new phase of work.

Resetting usage does not affect conversations or any other data.
